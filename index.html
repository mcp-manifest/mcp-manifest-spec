mcp-manifest.json

Autodiscovery and zero-config installation for MCP servers.

ironlicensing.com → manifest discovered → tool installed → connected

The Problem

Every MCP server is a snowflake installation. The protocol defines the handshake after connection, but says nothing about how to install, configure, or wire the client. Every README has a different JSON blob to copy-paste.

The Fix

Ship an mcp-manifest.json with your server. Enable autodiscovery with one line of HTML. Any MCP client can then set up your server from just a domain name.

Autodiscovery

Like RSS feeds. Add a <link> tag or serve at /.well-known/. Clients find it automatically.

Any Runtime

npm, pip, dotnet, cargo, Docker, binary. The manifest declares how to install — clients pick the right method.

Typed Config

API keys, paths, URLs — each typed so clients show the right UI. Secrets get masked automatically.

How It Works

1
Server author adds mcp-manifest.json to their repo and website
2
User types a domain name into their MCP client
3
Client discovers the manifest, installs the tool, prompts for config, writes settings

Enable Autodiscovery

HTML Add to your <head>:

<link rel="mcp-manifest" type="application/json" href="/mcp-manifest.json" />

API Or serve at a well-known URL:

GET https://your-site.com/.well-known/mcp-manifest.json

Minimal Manifest

{
  "$schema": "https://mcp-manifest.dev/schema/v0.1.json",
  "version": "0.1",
  "server": {
    "name": "my-server",
    "displayName": "My Server",
    "description": "What it does",
    "version": "1.0.0"
  },
  "install": [{
    "method": "npm",
    "package": "my-mcp-server",
    "command": "my-mcp-server",
    "priority": 0
  }],
  "transport": "stdio"
}
